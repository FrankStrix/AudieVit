AudieVit

AudieVit è un assistente vocale AI che gira interamente nel tuo browser, senza inviare dati a server esterni. Sfrutta le API Web Speech del browser per il riconoscimento vocale (STT) e la sintesi vocale (TTS), e si connette a un'istanza locale di Ollama per generare risposte intelligenti.

Privacy prima di tutto. Tutto il processing avviene in locale: il microfono, il browser e il modello AI sono sulla tua macchina. Nessun dato lascia il tuo computer.

Caratteristiche principali

  • Riconoscimento vocale — Trascrizione in tempo reale tramite Web Speech API
  • Risposte AI — Generazione tramite Ollama (modello llama2)
  • Sintesi vocale — Lettura ad alta voce delle risposte con voci naturali
  • Multi-lingua — Rilevamento automatico e risposta in IT, EN, FR, ES, DE
  • Streaming — Risposte in tempo reale carattere per carattere
  • Cronologia — Conversazioni salvate in SQLite con persistenza su localStorage
  • Interfaccia scura — UI moderna con tema scuro e animazioni fluide

Installazione

Prerequisiti

Assicurati di avere i seguenti strumenti installati:

Strumento Versione minima Note
Node.js 18.x Runtime JavaScript
npm 9.x Viene con Node.js
Ollama 0.1.x Server AI locale
Browser moderno — Chrome, Edge, Firefox o Safari (con supporto Web Speech)

Installare Ollama

Ollama è il server AI che esegue il modello linguistico in locale. Scaricarlo dalla pagina ufficiale o usare il gestore di pacchetti: 

# Linux (curl)
curl -fsSL https://ollama.ai/install.sh | sh

# macOS (Homebrew)
brew install ollama

# Windows — scarica il installer da https://ollama.ai

Dopo l'installazione, avvia il server e scarica il modello llama2:

ollama serve          # avvia il server in background
ollama pull llama2    # scarica il modello
Modelli alternativi. Puoi usare qualsiasi modello supportato da Ollama. Modifica il nome del modello nel file App/src/services/ollama.ts alla riga 15. Modelli consigliati: mistral, phi, llama3.

Scaricare AudieVit

Clona il repository e installa le dipendenze:

git clone https://github.com/FrankStrix/AudieVit.git
cd AudieVit/App
npm install

Configurare e avviare

Assicurati che Ollama sia in esecuzione, poi avvia il dev server: 

npm run dev

Apri il browser all'indirizzo http://localhost:1420.

Proxy API. Il server Vite è configurato per proxare le richieste /api/ollama/* verso localhost:11434 (la porta predefinita di Ollama). Consulta App/vite.config.ts per i dettagli.

Build di produzione

Per compilare l'applicazione per la distribuzione:

npm run build

L'output sarà nella cartella App/dist/. Puoi servirlo con qualsiasi server HTTP statico: 

npm run preview

Guida all'uso

Prima schermata (Welcome)

All'avvio dell'app compare la welcome page con:

  • Il logo e il nome AudieVit
  • Il tagline "Il tuo assistente vocale AI, locale e privato"
  • Una creator card con il profilo del creatore FrankStrix
  • Il pulsante "Iniziamo!" per entrare nell'app

Questa schermata viene mostrata una volta sola per sessione. Una volta entrati, il pulsante "Nuova chat" nell'header permette di iniziare una nuova sessione senza ripassare dalla welcome.

Iniziare una conversazione

Una volta dentro l'app, il flusso di base è semplice:

  1. Premi "Inizia Ascolto" — il browser chiederà il permesso di accedere al microfono
  2. Parla — pronuncia la tua domanda o comando
  3. Attendi la risposta — l'AI risponderà con un testo che viene letto ad alta voce
  4. Continua — la conversazione rimane aperta. Premi "Inizia Ascolto" per fare una nuova domanda
Consiglio: Parla chiaramente e a volume moderato per una trascrizione più accurata. Evita rumori di fondo eccessivi.

Durante l'ascolto, il testo trascritto in tempo reale appare in una barra nella parte inferiore della chat ("Sto Ascoltando...").

Riconoscimento vocale (STT)

AudieVit utilizza l'API Web Speech del browser, specificamente SpeechRecognition (con fallback webkitSpeechRecognition per browser basati su Chromium).

Caratteristiche dell'implementazione:

  • Riconoscimento continuo — rimane in ascolto finché non viene fermato manualmente o finché non riceve un risultato finale
  • Risultati intermedi — mostra il riconoscimento in tempo reale mentre parli
  • Auto-restart — se la connessione col microfono cade, il servizio tenta di riavviarsi automaticamente dopo 200 ms
  • Soglia di confidenza — i risultati finali con confidenza inferiore a 0.3 vengono scartati

Risposte AI (Ollama)

Il testo trascritto viene inviato a un'istanza locale di Ollama tramite una richiesta HTTP POST /api/ollama/api/generate. La richiesta viene proxyata da Vite a localhost:11434.

Il sistema invia a Ollama:

  • Un system prompt che istruisce il modello a rispondere nella lingua rilevata, in modo conciso, e fornisce la data/ora italiana corrente
  • Le ultime 3 domande dell'utente come contesto
  • La nuova domanda

La risposta viene streaming carattere per carattere e mostrata in tempo reale nell'interfaccia. Appena una frase è completa (termina con .!?), viene accodata alla sintesi vocale.

Sintesi vocale (TTS)

AudieVit usa l'API SpeechSynthesis del browser per leggere ad alta voce le risposte.

Caratteristiche:

  • Coda di frasi — le frasi vengono accodate e lette in sequenza, con una pausa di 120 ms tra una frase e l'altra
  • Sanificazione del testo — le entità HTML (&egrave;, &agrave;, ecc.) e la sintassi Markdown vengono convertiti o rimossi prima della lettura
  • Lingua appropriata — la sintesi usa la voce corrispondente alla lingua rilevata (it-IT, en-US, fr-FR, es-ES, de-DE)
  • Interruzione — se parli mentre l'assistente sta parlando, la sintesi viene interrotta e la nuova domanda viene processata

Gestione delle sessioni

AudieVit organizza le conversazioni in sessioni. Ogni sessione contiene una serie di messaggi (domande e risposte).

  • Cronologia — clicca su "Cronologia" nell'header per vedere l'elenco delle sessioni passate, ordinate dalla più recente
  • Caricare una sessione — clicca su una sessione nella cronologia per riaprirla
  • Nuova sessione — clicca "Nuova chat" per iniziare una nuova conversazione
  • Cancellare tutto — nella cronologia, il pulsante "Cancella tutto" elimina tutte le sessioni e i messaggi
Persistenza automatica. I messaggi vengono salvati automaticamente in localStorage tramite SQLite (sql.js). Il salvataggio avviene con debounce di 2 secondi dopo l'ultimo messaggio, e immediatamente per le operazioni critiche (cancellazione, cambio sessione).

Rilevamento della lingua

AudieVit rileva automaticamente la lingua del tuo parlato usando pattern regex. Le lingue supportate sono:

Lingua Codice Tag TTS
Italianoitit-IT
Ingleseenen-US
Francesefrfr-FR
Spagnoloeses-ES
Tedescodede-DE

L'inglese viene testato per primo (ha il pattern più esteso con oltre 150 parole chiave). Se non viene rilevato, si procede con gli altri pattern specifici per lingua (lettere accentate e parole comuni). Se nessun pattern matcha, l'italiano è la lingua predefinita.

Frontend (React + TypeScript + Vite)

L'applicazione è costruita con React 19 e TypeScript 5.8, bundlata con Vite 7. Lo stile è gestito con Sass (SCSS).

Il dev server Vite è configurato sulla porta 1420 con un proxy che inoltra le richieste /api/ollama/* a Ollama (localhost:11434), riscrivendo il path.

Servizi

La logica di business è suddivisa in moduli indipendenti nella cartella src/services/. Ogni servizio è un modulo TypeScript puro (nessuna dipendenza da React), il che li rende facilmente testabili e riutilizzabili.

speechRecognition.ts

Incapsula l'API SpeechRecognition del browser. Espone due funzioni:

FunzioneDescrizione
startListening(callbacks) Avvia l'ascolto. Accetta un oggetto con onFinal(text) e onInterim(text). Il riconoscimento è continuo e si riavvia automaticamente se la connessione cade.
stopListening() Ferma l'ascolto e rilascia le risorse.

ollama.ts

Gestisce la comunicazione con Ollama. Espone:

FunzioneDescrizione
generateResponse(question, onChunk?, history?) Invia una richiesta POST streaming a Ollama. Il parametro onChunk riceve il testo completo accumulato a ogni chunk. La funzione restituisce una Promise con la risposta completa.

La richiesta include un system prompt con la lingua rilevata e la data/ora italiana, più il contesto delle ultime 3 domande dell'utente.

textToSpeech.ts

Incapsula l'API SpeechSynthesis. Espone:

FunzioneDescrizione
speak(text, lang?) Legge il testo ad alta voce usando la lingua specificata (o automaticamente rilevata). Restituisce una Promise che si risolve quando la lettura è completa. Prima di parlare, sanifica il testo (rimuove HTML, markdown, emoji).
stopSpeaking() Interrompe immediatamente la sintesi vocale.

language.ts

Rilevatore automatico di lingua basato su espressioni regolari. L'inglese ha un pattern esteso di oltre 150 parole comuni. Per le altre lingue, combina lettere accentate e parole chiave.

FunzioneDescrizione
detectLanguage(text) Analizza il testo e restituisce un oggetto LangInfo con codice, nome italiano e tag TTS.

database.ts

Gestisce la persistenza tramite sql.js (SQLite compilato in WebAssembly). Il database risiede in memoria e viene salvato periodicamente su localStorage come stringa Base64.

Schema del database:

sessions
├── id           INTEGER PRIMARY KEY AUTOINCREMENT
└── created_at   TEXT NOT NULL

messages
├── id           INTEGER PRIMARY KEY AUTOINCREMENT
├── session_id   INTEGER NOT NULL (FK → sessions.id)
├── role         TEXT NOT NULL ('user' | 'assistant')
├── content      TEXT NOT NULL
└── created_at   TEXT NOT NULL

Funzioni esportate:

FunzioneDescrizione
initDatabase()Inizializza SQLite e carica i dati salvati
createSession()Crea una nuova sessione, restituisce l'ID
addMessage(sessionId, role, content)Aggiunge un messaggio a una sessione
getSessionMessages(sessionId)Restituisce tutti i messaggi di una sessione
getSessions()Restituisce tutte le sessioni con preview
deleteSession(sessionId)Elimina una sessione e i suoi messaggi
clearAllSessions()Elimina tutte le sessioni e i messaggi
persistNow()Salva immediatamente su localStorage

Proxy e networking

La configurazione di Vite (vite.config.ts) definisce un proxy per evitare problemi di CORS: 

server: {
  port: 1420,
  proxy: {
    '/api/ollama': {
      target: 'http://localhost:11434',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api\/ollama/, ''),
    },
  },
}

In pratica, una richiesta a /api/ollama/api/generate viene inoltrata a http://localhost:11434/api/generate.

Persistenza dei dati

AudieVit utilizza sql.js, una build di SQLite compilata in WebAssembly, per gestire la persistenza:

  1. All'avvio, il database viene caricato da localStorage (chiave audievit_v3_db)
  2. Se non esiste un salvataggio, viene creato un database vuoto con le tabelle
  3. Ogni messaggio aggiunto attiva un salvataggio con debounce di 2 secondi
  4. Le operazioni critiche (cancellazione, cambio sessione) salvano immediatamente
  5. Il database viene esportato in Base64 e salvato in localStorage
Attenzione: I dati sono salvati nel localStorage del browser. La cancellazione dei dati del browser comporta la perdita della cronologia. Non esiste sincronizzazione tra dispositivi.

API Reference

speechRecognition.ts

startListening(callbacks)

Avvia il riconoscimento vocale continuo. Il parametro callbacks è un oggetto con:

CallbackFirmaDescrizione
onFinal (text: string) => void Chiamato quando un risultato finale è disponibile (confidenza ≥ 0.3)
onInterim (text: string) => void Chiamato con i risultati provvisori in tempo reale

Il riconoscimento si riavvia automaticamente in caso di disconnessione (onend), finché non viene chiamato stopListening().

stopListening()

Interrompe il riconoscimento e rilascia l'istanza.

ollama.ts

generateResponse(question, onChunk?, history?)

ParametroTipoDescrizione
questionstringIl testo della domanda
onChunk(fullText: string) => voidCallback opzionale per lo streaming, chiamato con il testo accumulato
history{ role: string; content: string }[]Messaggi precedenti per il contesto (ultimi 3 user)

Restituisce Promise<string> con la risposta completa.

Il system prompt è generato automaticamente e include: lingua rilevata, data/ora italiana corrente, e l'istruzione di rispondere in modo conciso.

textToSpeech.ts

speak(text, lang?)

ParametroTipoDescrizione
textstringTesto da leggere ad alta voce
langLangInfoOpzionale. Se non specificato, la lingua viene rilevata automaticamente

Restituisce Promise<void> che si risolve quando la lettura termina.

Il testo viene sanificato prima della lettura: le entità HTML vengono convertite, la sintassi Markdown e le emoji rimosse.

stopSpeaking()

Interrompe immediatamente qualsiasi sintesi vocale in corso.

language.ts

Tipi

type LangInfo = {
  code: string;    // 'it' | 'en' | 'fr' | 'es' | 'de'
  name: string;    // 'italiano' | 'inglese' | ...
  tts: string;     // 'it-IT' | 'en-US' | ...
};

detectLanguage(text)

ParametroTipoDescrizione
textstringIl testo da analizzare

Restituisce LangInfo. L'ordine di rilevamento è: inglese → italiano → francese → spagnolo → tedesco → italiano (default).

database.ts

initDatabase()

Inizializza SQLite. Carica i dati da localStorage se presenti, altrimenti crea un database vuoto. Chiamare una sola volta all'avvio.

createSession()

Crea una nuova sessione con timestamp corrente. Restituisce l'id della sessione come number.

addMessage(sessionId, role, content)

Aggiunge un messaggio alla sessione specificata. role può essere 'user' o 'assistant'.

getSessionMessages(sessionId)

Restituisce un array di Message[] ordinati per id ascendente.

getSessions()

Restituisce un array di Session[] con la preview (primo messaggio utente), ordinati per id discendente.

deleteSession(sessionId)

Elimina la sessione e tutti i suoi messaggi. Salva immediatamente.

clearAllSessions()

Elimina tutte le sessioni e tutti i messaggi. Salva immediatamente.

persistNow()

Forza un salvataggio immediato su localStorage.

Tipi

interface Message {
  id: number;
  session_id: number;
  role: 'user' | 'assistant';
  content: string;
  created_at: string;
}

interface Session {
  id: number;
  created_at: string;
  preview: string;
}

Estendere AudieVit

Alcune idee per personalizzare e migliorare l'assistente:

  • Cambiare modello AI: modifica il campo model in ollama.ts (riga 15)
  • Aggiungere una lingua: aggiungi un nuovo pattern in language.ts seguendo la struttura esistente
  • Personalizzare l'UI: modifica le variabili CSS in styles.scss (palette colori, raggi, transizioni)
  • Integrare strumenti esterni: aggiungi chiamate API nel servizio Ollama o crea nuovi servizi

FAQ e risoluzione dei problemi

Il microfono non funziona

  • Assicurati di aver concesso il permesso al browser di accedere al microfono
  • Controlla che il microfono sia collegato e funzionante
  • Prova con un browser basato su Chromium (Chrome, Edge, Brave) — Firefox e Safari hanno limitazioni con l'API SpeechRecognition
  • Apri la console del browser (F12) per verificare eventuali errori

Ollama non risponde

  • Verifica che Ollama sia in esecuzione: ollama ps
  • Controlla che il modello sia stato scaricato: ollama list
  • Testa l'API direttamente: curl -X POST http://localhost:11434/api/generate -d '{"model":"llama2","prompt":"Ciao"}'
  • Assicurati che la porta 11434 non sia bloccata da un firewall
  • Verifica la configurazione del proxy in vite.config.ts

La sintesi vocale non funziona

  • Verifica che il browser supporti speechSynthesis — la maggior parte dei browser moderni lo supportano
  • Controlla che il volume del sistema non sia muto
  • Alcuni browser richiedono un'interazione utente prima di poter usare la sintesi vocale — clicca un pulsante prima di parlare

La cronologia viene persa

  • I dati sono salvati in localStorage — non cancellare i dati del browser
  • La modalità di navigazione in incognito/navigazione privata perde i dati alla chiusura della finestra
  • Se il database è corrotto, puoi cancellarlo aprendo la console ed eseguendo: localStorage.removeItem('audievit_v3_db')

Altri problemi

Se incontri altri problemi, apri una segnalazione su GitHub Issues includendo:

  • Il tuo browser e sistema operativo
  • Il contenuto della console del browser (F12 → Console)
  • La versione di Node.js e npm
  • La versione di Ollama e il modello usato
